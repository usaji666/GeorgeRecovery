# 治疗乔治腿伤

作者：vicky

内部 ID：`vicky.GeorgeRecovery`

## v0.3.2 测试版功能

- 乔治和哈维同时达到 8 心后，在雨天 9:00—18:00 进入乔治家触发剧情。
- 首段剧情使用真正的地图内过场：乔治与玩家面对面；前三段对话结束后，哈维直接在客厅门口出现，先向下一格、再向右三格，绕开玩家并停在两人下方，然后再开始对话。
- macOS 进入存档后保留星露谷原版棕色软件鼠标并隐藏系统箭头；不在标题和入场动画阶段修改任何显示状态。
- NPC 对话使用游戏原版头像与姓名框，不再把说话人的名字写进正文。
- 接受请求后立即启用哈维楼上的原冰箱：显示黄色感叹号并打开 36 格任务材料箱；支持分批提交并永久累计进度，收集完成后感叹号消失并恢复原版不可储物状态。
- 任务冰箱直接创建原版 `ItemGrabMenu` 36 格收纳菜单，不再调用只适用于已摆放地图箱子的 `Chest.checkForAction`，修复“任务材料箱菜单未能打开”。
- 为方便测试，当前只需要提交 1 个兔子的脚；哈维的接受任务台词已同步修改。
- 普通、银星、金星和铱星兔子的脚均可提交；菜单使用原版 Chest 存取回调，修复物品无法从背包放入的问题。
- 乔治第三句台词结束后增加 1 秒停顿，再播放开门声并让哈维入场。
- Mod 自动扣除符合条件的背包材料，并按存档保存累计进度。
- 拒绝请求会分别扣除乔治和哈维 50 好感，并永久关闭任务。
- 材料集齐后，在非节日 9:00—18:00 进入乔治家触发完成剧情。
- 完成康复剧情后，乔治每天 12:00—17:00 会换成专用站立行走精灵；17:00 后恢复原版轮椅精灵。
- 乔治康复后的下午日程以“在乔治家练习”和“陪艾芙琳活动”为主：春夏会陪艾芙琳去镇上，周一、周四和周六会陪她去社区中心，雨天与秋冬大部分时间留在家里，特殊就诊日和冬季夜市日也会与她同行。
- 只替换原版日常对话中直接抱怨双腿的 `Wed8` 和 `Tue10` 两组台词，改为积极的康复内容；其他日常台词保持原版。
- 站立精灵使用乔治原版头部特征和新制作的四方向 16 帧行走身体；原版头像不变。
- v0.3.1 重新按原版 NPC 的身体比例制作站立精灵：补出头部与衣领之间的颈部像素，收窄正面和背面的肩膀、躯干，并使用原版交替摆臂和分腿帧；两个迈步帧额外抬高 1 像素，使行走时产生清楚而克制的上下节奏。
- v0.3.2 将轮椅版头部上移到原版站立 NPC 的位置，完整保留乔治朝下时的脸、眼镜和下巴；颈部增加到两像素，腿部缩短两像素。雨天在家、诊所和冬季夜市日的站位也与艾芙琳至少间隔两格。
- 构建默认不自动部署到游戏，避免测试中的版本被意外装回。

## 开发与安装

项目需要 Stardew Valley、SMAPI 4.0+ 和 .NET SDK。

```bash
cd GeorgeRecovery
dotnet build
```

构建成功后会生成 `bin/Debug/net6.0/GeorgeRecovery 0.3.2.zip`，但不会自动复制到游戏。确认要测试时，手动解压到游戏的 `Mods/GeorgeRecovery`。

## 测试命令

在 SMAPI 控制台输入：

```text
george_recovery_status
```

它会显示任务阶段、材料进度、当前地点和新材料箱的格子坐标。

开发测试时还可以设置阶段：

```text
george_recovery_setstage notstarted
george_recovery_setstage collecting
george_recovery_setstage materials
george_recovery_setstage declined
george_recovery_setstage completed
```

该命令会直接改写当前存档内本 Mod 的任务数据，只应用在测试存档。

## 当前测试重点

1. 首段过场的乔治、玩家和哈维站位及行走路径是否自然。
2. 雨天进入 `JoshHouse` 时首段剧情是否只触发一次，人物头像是否正常。
3. 收集阶段点击哈维楼上的冰箱是否打开 36 格任务容器，完成后是否恢复原版行为。
4. 放入 1 个兔子的脚后是否正确扣除并立即完成材料收集。
5. 拒绝路线是否永久关闭任务。
6. 材料完成后是否能触发最终剧情。
7. 康复剧情完成后的下一个正常游戏日，12:00 乔治是否切换为站立形象并按新日程行走，17:00 是否恢复轮椅。
8. 八心周三与十心周二的两组腿伤台词是否变为积极内容，其他日常对话是否仍为原版。
